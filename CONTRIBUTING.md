# Contributing to Business AI Solutions Suite

We welcome contributions to expand and improve this suite of professional AI agents.

## Development Workflow

1.  **Fork** the repository.
2.  **Clone** your fork locally.
3.  **Create a Branch** for your feature or fix (`git checkout -b feature/amazing-agent`).
4.  **Implement** your changes.
5.  **Test** thoroughly using the provided validation scripts (if applicable) or manual execution.
6.  **Commit** your changes with clear messages (`git commit -m "feat: add vector database integration"`).
7.  **Push** to your fork (`git push origin feature/amazing-agent`).
8.  **Open a Pull Request**.

## Code Style

- Follow PEP 8 guidelines for Python code.
- Ensure all new dependencies are added to `requirements.txt`.
- Document new agents in `README.md` following the existing format.

## Reporting Issues

Please use the GitHub Issue tracker to report bugs or request features. Provide detailed reproduction steps for any bugs.
